# Anti-SEO Researcher

**反SEO反水军深度消费决策研究工具 V5**

> 在一个搜索结果被SEO和水军严重污染的互联网环境中，帮你找到真实的消费反馈，做出靠谱的购买决策。

---

## 它解决什么问题

当你想买一个商品（手机、奶粉、人体工学椅、防晒霜……）时，你去搜索引擎搜，看到的大多是：

- 知乎高赞回答 —— 很多是品牌投放的软文
- 小红书种草帖 —— 大量是广告合作
- 什么值得买推荐 —— 带返利链接的利益相关内容
- B站测评视频 —— 品牌赞助的"恰饭"内容

而**真正有价值的消费反馈**藏在这些地方：

- 京东/淘宝的**追评**（用了半年后的真实感受）
- 小红书种草帖下面的**评论区**（"我也买了，翻车了"）
- V2EX/Chiphell 等小众社区的**长文吐槽**
- 知乎回答下面的**反驳评论**

这个工具做的事情就是：**自动化地从中文互联网的各个角落搜集真实用户反馈，用算法过滤掉软文和水军，对候选商品进行多维度评分，最终生成一份带有推荐/避坑结论的调查报告。**

## 核心特性

### 三层数据源架构（V5）

| 数据源层级 | 来源 | 为什么有价值 | 可信度权重 |
|-----------|------|------------|-----------|
| **L1 电商追评** | 京东/淘宝/拼多多评论（间接搜索） | 花了真金白银的买家，长期使用后的追评 | 0.85 |
| **L2 评论区** | 小红书/知乎评论区（间接搜索） | 种草帖下面的"拔草"评论，反驳软文的真实声音 | 0.75 |
| **L3/L4 社区帖** | V2EX / Chiphell / NGA / 贴吧等 | 发烧友的深度体验帖和横向对比 | 按平台动态调整 |

> 电商评论和社交评论区是动态加载内容，搜索引擎无法直接索引。V5 通过**间接搜索策略**（搜索"追评汇总"、"差评合集"、"评论搬运帖"等二手内容）来获取这些高价值数据。

### 双层可信度评分

- **第一层：正则规则引擎** —— 100+ 条规则识别软文特征（营销话术、零缺点描述、购买链接、好评返现等）和真实反馈特征（使用时长、缺点描述、售后经历等）
- **第二层：AI 语义分析** —— 对正则评分处于灰色地带（30-85分）的内容，调用 AI 做5维度精细判断（语气真实性、论述逻辑、细节丰富度、情感一致性、利益关联），节约约40-50% AI调用成本

### 品类自适应

不同品类的消费决策关注点完全不同。工具通过 `category_profile` JSON 驱动所有品类差异化行为：

- **奶粉**：关注配方成分、安全召回事件，安全封顶阈值最严格
- **手机**：关注SOC性能、续航、发热，搜索覆盖数码论坛
- **人体工学椅**：关注久坐体验、网布耐久性，搜索覆盖Chiphell/V2EX
- **防晒霜**：关注成分安全性、肤感体验，搜索覆盖小红书/美妆论坛

### 安全事件筛查

对每个推荐商品的品牌执行安全事件专项搜索。双层分级体系：

- **通用安全层**：召回、致死、下架、诉讼、监管处罚
- **品类安全层**：如奶粉的三聚氰胺、电子产品的电池爆炸、椅子的气压棒爆炸

安全封顶规则确保存在严重安全问题的商品不会被推荐，无论其他维度得分多高。

## 这是一个 AI Skill

本项目是一个 **AI Skill**（技能包），不是独立运行的程序。它需要运行在支持 Skill 的 AI 编码助手中（如 [CodeBuddy](https://codebuddy.com)、[Cursor](https://cursor.com)、[Windsurf](https://windsurf.com) 等）。

**为什么是 Skill 而不是独立 Agent 或 MCP？**

- 工具的搜索能力依赖宿主 AI 提供的 `web_search` 和 `web_fetch`，无需额外的 API Key
- 工具的 AI 分析能力由宿主 AI 直接执行，无需单独的 LLM 接口
- 11步工作流已在 SKILL.md 中完整定义，不需要 Agent 的动态规划能力
- 零基础设施成本，拷贝文件夹即可使用

## 安装

### 方法一：直接拷贝（推荐）

将整个 `anti-seo-researcher` 文件夹拷贝到你的 AI 编码助手的 Skill 目录下：

```
# CodeBuddy 示例
cp -r anti-seo-researcher/ your-project/.codebuddy/skills/

# Cursor 示例
cp -r anti-seo-researcher/ your-project/.cursor/skills/
```

### 方法二：Git Clone

```bash
git clone https://github.com/your-username/anti-seo-researcher.git
# 然后将目录拷贝到你的 AI 编码助手的 Skill 目录下
```

### 环境要求

- Python >= 3.6（所有脚本仅使用 Python 标准库，无第三方依赖）
- 支持 Skill 的 AI 编码助手（需提供 `web_search` 和 `web_fetch` 工具）

## 使用方法

安装完成后，在 AI 编码助手的对话中直接用自然语言描述你的购买需求即可：

```
用户：帮我调研一下3000元以内的人体工学椅，主要打游戏用，每天坐6小时以上

用户：想给宝宝买奶粉，预算300-350一罐，6个月大，主要关注安全性和消化吸收

用户：预算5000左右的手机，主要拍照和日常使用，不玩游戏
```

工具会自动：

1. 确认你的需求（品类、预算、使用场景）
2. 生成品类自适应配置
3. 从电商评论、社交评论区、技术社区三层数据源搜索真实反馈
4. 过滤软文和水军内容
5. 对候选商品进行安全事件筛查
6. 执行多维度评分
7. 生成完整的调研报告（Markdown格式）

## 工作流（7步 + 4中间步）

| 步骤 | 名称 | 执行方式 | 说明 |
|------|------|---------|------|
| **1** | 需求确认 | AI 对话 | 确认品类、预算、使用场景（最多3个问题） |
| **1.5** | 品类分析 | AI 生成 JSON | 生成 `category_profile`，驱动后续所有步骤 |
| **2** | 三层搜索 | AI + 脚本 | L1电商评论 + L2评论区 + L3社区帖 |
| **3** | 内容抓取 | AI + web_fetch | 抓取有价值帖子的完整正文 |
| **3.5** | 参数提取 | AI | 从内容中提取商品结构化参数表 |
| **4** | 动态深挖 | 脚本 | 对高频型号执行负面长尾搜索 + 电商追评深挖 |
| **4.5** | 安全搜索 | 脚本 | 对候选品牌搜索安全事件（召回/毒素/爆炸等） |
| **5** | 可信度评估 | 脚本 + AI | 正则粗筛 → AI精判（灰色地带） |
| **5.5** | 冲突仲裁 | 脚本 | 不同来源矛盾评价的加权裁决 |
| **6** | 多维评分 | 脚本 | 按品类维度/权重打分 + 安全封顶 |
| **7** | 报告生成 | 脚本 | 输出 Markdown 报告 |

## 评分体系

### 可信度评分（过滤软文）

每条搜索结果获得 0-100 的可信度评分：

- **85-100**：高度可信（明确的长期使用反馈、有具体缺点描述）
- **60-84**：基本可信（有一定参考价值）
- **30-59**：灰色地带（触发 AI 语义分析精判）
- **0-29**：疑似软文/水军（自动过滤）

### 商品综合评分（推荐结论）

| 综合分 | 结论 |
|--------|------|
| >= 70 | 推荐 |
| 55-69 | 有条件推荐 |
| 40-54 | 谨慎选择 |
| < 40 | 避坑 |

### 安全封顶规则

即使一个商品在其他维度评分很高，如果存在安全问题，综合分会被强制封顶：

| 品类类型 | 封顶阈值 | 说明 |
|---------|---------|------|
| food（食品/奶粉） | 30分 | 最严格，安全问题直接避坑 |
| personal_care（个护/美妆） | 25分 | 成分安全敏感 |
| electronics（电子产品） | 20分 | 电池/充电安全 |
| durable_goods（耐用品） | 15分 | 结构安全 |

## 关键规则

1. **先确认需求再搜索** —— 预算、品类、使用场景三者缺一不可
2. **先生成 category_profile 再搜索** —— 它是所有步骤的基石
3. **电商评论层不可跳过** —— 占总搜索量 >= 15%
4. **评论区层不可跳过** —— 占总搜索量 >= 15%
5. **负面搜索占比 >= 40%** —— 主动搜负面，搜不到说明深度不够
6. **安全事件必须搜** —— 推荐品牌必须过安全筛查
7. **追评 > 开箱** —— 长期使用追评的参考价值远高于开箱评测
8. **评论区 > 帖子正文** —— 评论区的真实反馈优先于帖子正文结论
9. **交叉验证是核心** —— 商业评测说好 + 小众论坛抱怨 → 信后者
10. **知乎万赞要警惕** —— 高赞不等于真实

## 技术细节

### 零外部依赖

- 所有 Python 脚本仅使用标准库（`json`, `re`, `urllib`, `ssl`, `argparse` 等）
- 无需 `pip install` 任何包
- 无需任何 API Key（搜索通过宿主 AI 的 `web_search` 工具执行）

### 间接搜索策略

电商评论是动态加载内容，搜索引擎无法直接索引。V5 的核心创新是通过搜索"评论搬运帖"来间接获取：

```
"[商品型号] 京东评论 追评 真实 差评"     → 搜索评论搬运/汇总帖
"[商品型号] 追评 用了半年 一年 长期"      → 搜索追评专项内容
"[商品型号] 小红书 评论区 真实 翻车"      → 搜索评论区反馈
```

### AI 成本控制

AI 语义分析仅对正则评分 30-85 分的"灰色地带"内容触发，配合本地 JSON 文件缓存避免重复分析，节约约 40-50% 的 AI 调用成本。

## 示例输出

工具最终输出的调研报告是一个 Markdown 文件，包含以下章节：

1. **需求回顾** —— 品类、预算、使用场景
2. **安全风险警告** —— 如有安全事件，按来源类型分组展示
3. **候选商品评分表** —— 动态表头（由品类维度决定），包含综合分和结论
4. **调查结论** —— 推荐/避坑/中性三类，每个结论附真实用户引用和来源链接
5. **数据源分布统计** —— L1/L2/L3-L4 各层占比
6. **数据透明度** —— 搜索总量、过滤量、AI分析覆盖率、平台分布
7. **免责声明**
